## Methodology

We benchmarked W0rkTree against Git 2.43 on a variety of common operations using repositories of different sizes. These benchmarks were run on a standard 16-core developer workstation with NVMe SSDs to simulate real-world monorepo conditions.

## Results

### Storage Architecture: BLAKE3 & FastCDC

W0rkTree completely abandons Git's zlib-compressed packfiles in favor of a modern chunked storage architecture. By combining **FastCDC** (Fast Content-Defined Chunking) with **BLAKE3** hashing, W0rkTree achieves massive deduplication across large binaries and text files, reducing storage overhead while keeping object integrity verification computationally light.

### Clone Operations
W0rkTree's parallel chunk transfer reduces clone times by up to 5x on large repositories. By lazily loading large chunks and verifying integrity in parallel using BLAKE3, developers can start working instantly without waiting for a monolithic download.

### Snapshot Performance
Thanks to the local background worker, W0rkTree snapshots are nearly instantaneous — changes are pre-hashed and synced in the background as you work.

### Branch Switching
Tree isolation means branch switching only affects the relevant tree, not the entire repository.

## Conclusion

W0rkTree's Rust-powered engine delivers significant performance improvements across all common version control operations.
