W0rkTree operates using a two-runtime model: the foreground CLI (`wt`) and a background worker daemon. The background process is responsible for real-time synchronization, staging snapshots, live mirroring, and garbage collection without blocking your terminal.

## wt worker start

Start the W0rkTree background worker process.

```bash
wt worker start [--detached]
```

| Option | Description |
|---|---|
| `--detached` | Run the worker in the background (default behavior) |

<Callout type="info">
By default, the worker starts automatically when you run `wt track start` or when an operation requires background synchronization. You typically only need to start it manually for debugging.
</Callout>

## wt worker stop

Stop the currently running background worker process safely.

```bash
wt worker stop
```

This command signals the worker to finish any in-flight synchronizations and terminate gracefully.

## wt worker restart

Restart the background worker process.

```bash
wt worker restart
```

## wt worker status

Check the current status of the background worker.

```bash
wt worker status
```

**Example:**

```bash
$ wt worker status
Status: Running
PID: 14032
Uptime: 4h 23m
Active Tasks: 
  - Syncing tree 'frontend' (staged changes)
  - Live Mirror Mode: remote 'origin'
```

## wt worker logs

View or tail the logs generated by the background worker process.

```bash
wt worker logs [--follow] [--lines <n>]
```

| Option | Description |
|---|---|
| `--follow`, `-f` | Output appended data as the file grows |
| `--lines <n>`, `-n` | Output the last `n` lines (default: 50) |

<Callout type="warn">
If you encounter sync issues, checking the worker logs with `wt worker logs -f` is the best first step for troubleshooting.
</Callout>
