W0rkTree allows you to export a snapshot of your project as a compressed archive, making it easy to share code, distribute releases, or create backups without including the entire version control history (`.wt` directory).

## wt archive

Create an archive of files from a named tree, branch, or specific snapshot.

```bash
wt archive [<snapshot-or-branch>] [--format <tar|zip|tar.gz|tar.zst>] [--output <file>]
```

| Option | Description |
|---|---|
| `<snapshot-or-branch>` | The snapshot to export (defaults to the current `HEAD`) |
| `--format` | Archive format: `tar`, `zip`, `tar.gz`, `tar.zst` (default: inferred from `--output`, or `tar` if stdout) |
| `--output <file>`, `-o` | Write the archive to the specified file instead of stdout |
| `--tree <name>` | Export from a specific tree |
| `--prefix <path>/` | Prepend a prefix path to all files in the archive (e.g., `project-v1.0/`) |

**Examples:**

Create a gzipped tarball of the current tree:

```bash
$ wt archive -o project-v1.0.tar.gz --prefix project-v1.0/
✓ Created project-v1.0.tar.gz (14.2 MB)
```

Create a zip archive of the `release/2.0` branch from the `backend` tree:

```bash
$ wt archive release/2.0 --tree backend -o backend-v2.0.zip
✓ Created backend-v2.0.zip (8.5 MB)
```

## License Compliance Integration

The `wt archive` command integrates with W0rkTree's built-in license compliance engine to prevent the accidental distribution of code that violates your organization's legal policies.

```bash
wt archive -o release.zip --enforce-licenses
```

<Callout type="info">
When `--enforce-licenses` is used, W0rkTree audits the snapshot against the policies defined in `.wt-tree/license.toml`. If forbidden licenses are detected, or if dependencies lack license metadata, the archive process will be blocked.
</Callout>

```bash
$ wt archive -o release.zip --enforce-licenses
Auditing licenses...
❌ VIOLATION: Dependency 'legacy-util' uses forbidden license 'GPL-3.0'
Archive aborted due to license policy violations.
```

## Reproducible Builds

Archives generated by W0rkTree are deterministic. For any given snapshot, `wt archive` will consistently produce an archive with the exact same cryptographic hash (assuming the same format and prefix). This ensures reliable, verifiable, and reproducible build pipelines.
