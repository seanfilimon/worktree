## What is W0rkTree?

**W0rkTree** is a next-generation version control system built from the ground up in Rust. It reimagines how teams collaborate on code by introducing a **two-runtime architecture (bgprocess + server)**, **nested trees**, **staged snapshot visibility**, **multi-tenancy**, **declarative access**, and **license compliance** — all without sacrificing the speed and reliability modern engineering teams demand.

Unlike traditional version control systems that treat a repository as a single flat workspace, W0rkTree organizes code into **isolated trees** — independent workspaces that can have their own branches, permissions, and history while still living within a unified codebase. This means frontend, backend, infrastructure, and platform teams can all work independently without stepping on each other's toes.

<Callout type="info">
  W0rkTree is currently in **alpha**. APIs and commands may change between releases. We welcome feedback and contributions as we work toward a stable v1.0 release.
</Callout>

## Why W0rkTree?

If you're already using Git, you might wonder what W0rkTree brings to the table. Here's how they compare:

| Capability | Git | W0rkTree |
|---|---|---|
| **Repository model** | Single flat workspace | Nested isolated trees |
| **Change tracking** | Manual `git add` / `git commit` | Automatic background daemon |
| **In-flight visibility** | None | Staged snapshots |
| **Multi-tenancy** | None (platform-level) | Protocol-native |
| **Access control** | None (platform-level) | Declarative TOML, RBAC + ABAC |
| **License enforcement**| None | Per-path SPDX, server-enforced |
| **Large files** | Requires LFS | Native chunked storage |
| **History safety** | Rebase, force push | Append-only, no rewrite |
| **Team isolation** | Requires conventions (CODEOWNERS, paths) | First-class tree permissions |
| **Cross-team coordination** | Manual merge choreography | Linked branches with atomic merges |
| **Dependency management** | External tools (submodules, monorepo tools) | Built-in dependency graph with TODO generation |
| **Project management** | External tools (Jira, Linear, etc.) | Integrated PM with task attribution |
| **Performance** | C, shell scripts | Pure Rust, content-addressable store |
| **Git compatibility** | Native | Full import/export, push/pull support |

W0rkTree isn't a Git replacement — it's a **Git evolution**. You can import existing Git repositories, continue to push and pull from Git remotes, and migrate at your own pace.
