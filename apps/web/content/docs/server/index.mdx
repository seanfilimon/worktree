The **W0rkTree Server** (`worktree-server`) is the remote central daemon that manages repositories across multiple tenants, enforces IAM and licensing rules, and handles client synchronization. 

<Callout type="info">
  **Server vs. Background Process:** Do not confuse the central W0rkTree Server with the local Background Process (`bgprocess`). The local background process watches your filesystem and automatically tracks changes on your machine, while the Server is the remote authority that manages collaboration, access control, and synchronization between multiple users.
</Callout>

## Overview

The server operates as a long-running daemon that:

- **Manages Multi-Tenancy:** Provides logical and physical isolation between different organizations and trees.
- **Enforces Access Control (IAM):** Evaluates declarative `.wt-access.toml` policies and RBAC on every operation.
- **Enforces License Compliance:** Validates SPDX licenses and grants to prevent incompatible code merges.
- **Aggregates Staged Snapshots:** Streams real-time in-flight changes between connected clients.
- **Handles Merge Requests:** Manages the lifecycle and code review process for branch merges.
- **Handles Synchronization:** Facilitates the delta-compressed sync protocol between local instances and the central store.
