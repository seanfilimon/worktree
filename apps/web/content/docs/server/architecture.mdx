## Overview

The W0rkTree server is built on a **multi-tenant, event-driven architecture** that enables teams to work independently while maintaining a unified codebase. Every design decision prioritizes: **correctness first**, **speed by default**, and **collaboration without friction**.

## Threading Model

The server uses `tokio` as its async runtime with a multi-threaded executor:

- **Main thread** — accepts connections, manages lifecycle
- **Worker pool** — handles client requests (configurable, default: CPU cores)
- **Watcher thread** — dedicated thread for filesystem event collection
- **Background tasks** — garbage collection, cache eviction, metric aggregation

## Data Flow

### Local Operations

```
Working Directory → Filesystem Watcher → Event Engine → Snapshot Builder → Object Store → Branch Pointers
```

All local operations are **lock-free for reads** and use **per-tree write locks** for mutations.

### Sync Operations

```
Object Store (local) → Delta Calculator → Compression → Wire Protocol → Server (remote) → Object Store (remote)
```

The sync protocol is **incremental** — only objects the remote doesn't have are transferred.
