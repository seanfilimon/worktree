The **W0rkTree Protocol** (`worktree-protocol`) defines the canonical data model, serialization formats, and wire protocol used by all W0rkTree components. It is the foundational crate with zero external dependencies beyond `serde`.

<Callout type="info">
The protocol is versioned independently from the CLI and server. See [Protocol Versioning](/docs/protocol/versioning) for details.
</Callout>

## Overview

The protocol specification covers:

- **Two-Runtime Model** — the async interaction between the local background process and the remote server
- **Staged vs. Pushed State** — the distinct data models for real-time uncommitted changes vs. permanent immutable history
- **IAM Protocol** — declarative access control, role definitions, and scope resolution (tenant, tree, path)
- **Object Model** — the core object types (including Manifests, Deltas, Tags) and their BLAKE3-addressed storage
- **Tree Structure** — how nested trees work with independent histories, tree-scoped configurations, and access overrides
- **Wire Format** — `b"WT01"` envelope, Bincode serialization, and zstd compression for client-server communication
- **Operation Semantics** — how diffs, merges, and snapshots behave
