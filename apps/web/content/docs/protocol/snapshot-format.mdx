## Overview

A **snapshot** is an immutable point-in-time capture of a tree's state. Snapshots are the building blocks of history in W0rkTree.

## Format

```json
{
  "id": "wt:a3f2b1c8d4e5f6a7b8c9d0e1f2a3b4c5",
  "tree_root": "wt:e7d1f2a4b5c6d7e8f9a0b1c2d3e4f5a6",
  "parents": [
    "wt:b9c3d5e6f7a8b9c0d1e2f3a4b5c6d7e8"
  ],
  "author": {
    "name": "Jane Developer",
    "email": "jane@example.com"
  },
  "committer": {
    "name": "Jane Developer",
    "email": "jane@example.com"
  },
  "timestamp": "2025-01-15T10:30:00Z",
  "message": "Add navigation component",
  "tree_scope": "frontend",
  "metadata": {
    "auto_snapshot": false,
    "linked_branches": [],
    "reverts": "wt:c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9",
    "tags": ["v1.2.0"],
    "dependencies": {
      "backend": "wt:d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9"
    }
  }
}
```

## Fields

| Field | Type | Description |
|---|---|---|
| `id` | Hash | BLAKE3 hash of the snapshot content |
| `tree_root` | Hash | Reference to the root tree object |
| `parents` | Hash[] | Parent snapshot(s) — one for normal commits, two for merges |
| `author` | Identity | Who wrote the changes |
| `committer` | Identity | Who created the snapshot (may differ for auto-snapshots) |
| `timestamp` | DateTime | UTC timestamp of creation |
| `message` | String | Human-readable description |
| `tree_scope` | String | Which tree this snapshot belongs to |
| `metadata` | Object | Additional metadata (auto-snapshot flag, linked branches, reverts, tags, dependencies) |

## Extended Metadata

The `metadata` object stores flexible, extensible data about the snapshot's context:

- **`auto_snapshot`**: Indicates if the snapshot was generated by the continuous tracking daemon (staged) rather than explicitly committed.
- **`linked_branches`**: Tracks branches in other trees that were atomically modified alongside this one.
- **`reverts`**: The hash of a previous snapshot that this snapshot is reverting, providing explicit revert history.
- **`tags`**: Associated tags bound to this snapshot state.
- **`dependencies`**: A map of tree dependencies and the exact snapshot hashes they resolve to at the time this snapshot was created.

## History

Snapshots form a directed acyclic graph (DAG) through parent references. Each branch points to the latest snapshot in its lineage.

<Callout type="info">
Auto-snapshots created by the tracking daemon have `auto_snapshot: true` in their metadata. They can be squashed before sharing.
</Callout>
